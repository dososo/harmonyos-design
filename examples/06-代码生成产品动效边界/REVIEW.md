# 示例 6：代码生成产品动效的适用边界

## 问题

“动画审视 Skill 是否能用于通过某个代码工具生成视频？”

## 结论

不能只凭工具名称回答。先判断作品是否由可检查的状态、几何、时间和代码控制。

| 场景 | 主 Skill | Vocabulary | 说明 |
| --- | --- | --- | --- |
| ArkUI 交互界面 | 强适用 | 强适用 | 核心范围 |
| ArkUI Canvas 产品演示 | 适用 | 强适用 | 可审视时间、几何、排版和状态 |
| 代码生成的非交互产品视频 | 有限适用 | 适用 | 可复用节奏、连续性、排版；不套用点击反馈 |
| 通用剪辑、实拍、叙事视频 | 不适用 | 有限 | 需要镜头、剪辑、声音和叙事体系 |
| 黑盒生成式视频 | 不适用 | 有限 | 只能审视输出，难以映射到代码修复 |

## 第三方视频工具边界

本项目没有找到足以确认某第三方视频工具具体产品身份与能力的可靠官方资料，因此：

- 不声明兼容；
- 不把它写进 API 映射；
- 只在获得项目、代码或官方说明后做最小实验；
- 通用视频镜头语言仍不触发 `harmonyos-design`。

## 最小实验

```yaml
question: 该工具是否输出可控代码动画？
hypothesis: 若能控制状态、几何和时间，则部分运动规则可迁移
fixture: 10 秒产品状态切换
inputs: 代码、输出视频、时间轴
checks: 连续性、节奏、排版、状态真实性
exclusions: 点击命中、手势跟手、焦点和无障碍交互
```
