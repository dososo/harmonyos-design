# 示例 4：永恒原则与版本化视觉语言

## 情境

团队看到最新平台材质趋势后，准备把半透明、大圆角、强模糊和弹性错峰写成所有产品的永久规范。

## Findings

| 优先级 | 规则 ID | 问题 | 影响 |
| --- | --- | --- | --- |
| MAJOR | `HD-FOUNDATION-005` | 把当前视觉表现写进长期基础原则 | 系统版本变化后知识失真，也会覆盖产品自身语境 |
| MAJOR | `HD-FOUNDATION-004` | 用统一“时髦感”替代任务、风险与产品人格 | 金融、生产力、车机等界面可能被过度装饰 |
| MINOR | `HD-FOUNDATION-003` | 没有标明哪些参数只是 House Style | Agent 容易把偏好误称官方要求 |

## 正确分层

```text
Evergreen Foundation
- 状态真实
- 可读性
- 空间关系
- 可打断
- 无障碍

Versioned Platform Layer
- 当前材质
- 当前组件默认
- 当前 API 与设备能力

Project Overlay
- 品牌人格
- 风险等级
- 产品专用 Token
```

## 结论

**有条件通过。** 新视觉语言可以采用，但必须作为版本化平台层，并受到状态真实性、可读性、可访问性和产品语境约束。
